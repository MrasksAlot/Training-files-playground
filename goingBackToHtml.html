    given to the browser on how it should decode the image.
  • sync: Decode the image synchronously for atomic presentation with other content.
  • async:Decode the image asynchronously to reduce delay in presenting other content.
  • auto:Default mode, which indicates no preference for the decoding mode. The browser decides what is best for the user.

The preload attribute allows the author to provide a hint to the browser about when to load
the video The preload attribute is ignored if autoplay is present.

  • auto:The author thinks that the browser should load the entire video when the page loads
  • metadata:The author thinks that the browser should load only metadata (info about media) when the page loads
  • none:The author thinks that the browser should NOT load the video when the page loads
